[1]상속 관계에서의 생성자
- 생성자란?
- 생성자의 특징(클래스명과 동일, 리턴값이 없고, 오버로딩 가능)
- super(); : (모든 생성자 안에는)부모의 기본 생성자를 호출하는 코드!
- 매개변수 있는 생성자를 호출하기 위해서는, 객체();에 값을 넣어야한다.
- **자식 클래스 객체 생성 시, "생성자" 첫 라인에 부모클래스의 기본 생성자 호출!(안보임. 써줘도 됌!)

질문1) 서브 클래스의 인스턴트가 생성될 때 서브 클래스의 생성자와 수퍼 클래스의 생성자 모두 실행되는가?
 맞아욤!
질문2) 서브 클래스의 인스턴트가 생성될 때 서브 클래스의 생성자와 수퍼 클래스의 생성자의 실행 순서는 어떻게 되는가?
 호출은 서브 -> 수퍼 순이지만, 상위 클래스를 호출하는 super();가 가장 첫 번째 줄에 명시되어 있기 때문에, 수퍼클래스 부터 실행된다.

[2]메소드 내부 객체 변수, 생성자 호출 메소드
 1)메소드 내부 객체 변수
 - this : 메소드 안에서 객체를 나타내는 객체 변수이자, 메소드를 호출한 객체의 주소를 가지고 있다.
 - super : 메소드 안에서 상위 클래스 객체를 나타내는 객체 변수

 2) 생성자 호출 메소드
 - 생성자 안에서 다른 형태의 생성자 호출 가능
 - this() : 현재 클래스의 생성자 호출
 - super() : 부모 클래스의 생성자 호출
 - 생성자 : new를 이용하여 메모리 할당이 끝난 후 메모리를 초기화하는 역할
